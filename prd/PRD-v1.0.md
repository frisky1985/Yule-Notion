## 一、产品概述

### 1.1 产品定位

**予乐（Yule Notion）** 是一款面向个人的类 Notion 笔记本应用，支持自由部署和离线使用。核心价值：**你的笔记，完全由你掌控。**

### 1.2 目标用户

- 个人知识工作者（开发者、设计师、研究者等）
- 需要自部署、数据私有化的用户
- 对离线笔记有刚需的用户

### 1.3 产品目标

| 目标 | 量化指标 |
|------|----------|
| 核心编辑体验可用 | 支持至少6种块类型，编辑延迟 < 100ms |
| 离线可用 | 断网后可正常新建/编辑页面，联网后自动同步 |
| 自部署门槛低 | Docker Compose 一键部署，5分钟内启动 |
| 桌面端可用 | Electron 打包，支持 macOS / Windows / Linux |

## 二、用户场景与用例

### UC-01：创建和编辑笔记

**场景：** 用户打开应用，在侧边栏新建页面，输入标题后开始用富文本编辑器编写内容，插入代码块、引用等。

**步骤：**
1. 点击侧边栏「+ 新建页面」
2. 输入页面标题（自动保存）
3. 在编辑区使用工具栏插入块：标题（H1-H3）、无序列表、有序列表、代码块、引用、图片
4. 内容实时保存

**验收：** 页面标题和内容实时保存，无手动保存按钮；块类型切换流畅。

---

### UC-02：组织页面层级

**场景：** 用户通过拖拽或操作菜单，将页面组织成树形层级结构。

**步骤：**
1. 在侧边栏树形目录中右键页面，选择「添加子页面」
2. 或拖拽页面到另一个页面上方/下方/内部
3. 树形目录实时更新

**验收：** 支持3级嵌套；拖拽后层级正确更新；展开/折叠正常。

---

### UC-03：块编辑与拖拽排序

**场景：** 用户在页面中，通过输入 `/` 调出块类型菜单，切换块类型；通过拖拽调整块顺序。

**步骤：**
1. 在编辑器空行输入 `/`，弹出块类型选择菜单
2. 选择目标块类型（如代码块），当前块转换
3. 鼠标悬停块左侧出现拖拽手柄，拖动块到目标位置
4. 松开鼠标，块重新排列

**验收：** `/` 命令菜单弹出且可筛选；拖拽排序视觉反馈清晰；排序结果持久化。

---

### UC-04：全文搜索 + 标签

**场景：** 用户需要快速找到之前写的笔记，通过搜索栏输入关键词，或按标签筛选。

**步骤：**
1. 点击顶部搜索栏（或 `Ctrl/Cmd + K` 快捷键）
2. 输入关键词，实时显示匹配的页面列表（标题 + 内容摘要高亮）
3. 点击结果跳转到对应页面
4. 在页面设置中添加/管理标签，通过标签筛选页面

**验收：** 搜索结果在 500ms 内返回；关键词在搜索结果中高亮；标签支持增删和多标签筛选。

---

### UC-05：导出笔记

**场景：** 用户需要将页面内容导出为文件，分享给他人或备份。

**步骤：**
1. 打开页面，点击工具栏「导出」按钮
2. 选择导出格式（Markdown / PDF）
3. 文件下载到本地

**验收：** Markdown 导出保留标题层级、代码块、引用格式；PDF 导出排版清晰、中文不乱码。

---

### UC-06：离线使用

**场景：** 用户在没有网络的环境下打开应用，继续编辑笔记。恢复网络后，修改自动同步。

**步骤：**
1. 断网状态下打开应用
2. 应用正常加载（从本地缓存/IndexedDB）
3. 新建/编辑页面，内容保存到本地
4. 恢复网络，应用自动检测并同步本地变更到服务端
5. 同步状态有明确 UI 提示

**验收：** PWA 可安装；离线时应用完整可用；同步冲突时采用"最后写入优先"策略并提示用户。

---

### UC-07：Electron 桌面端使用

**场景：** 用户在桌面端使用予乐，获得原生应用体验。

**步骤：**
1. 下载对应平台安装包（macOS / Windows / Linux）
2. 安装后打开，登录账号
3. 功能与 Web 端一致

**验收：** 安装包可正常安装运行；离线功能可用；系统托盘最小化。

## 四、非功能需求

### 4.1 性能

| 指标 | 要求 |
|------|------|
| 页面首次加载（有网络） | < 3s（常规宽带） |
| 页面首次加载（离线） | < 1.5s |
| 编辑器输入延迟 | < 50ms（本地）；< 200ms（网络保存） |
| 搜索响应时间 | < 500ms（1000篇文档内） |
| 页面切换 | < 300ms |

### 4.2 安全

| 项目 | 要求 |
|------|------|
| 认证 | JWT Token；密码 bcrypt 加密存储 |
| 授权 | 用户只能访问自己的数据（数据隔离） |
| 文件上传 | 图片格式校验（jpg/png/gif/webp）；大小限制 5MB |
| API | RESTful + JWT Bearer Token；无敏感信息暴露在 URL 中 |

### 4.3 兼容性

| 平台 | 要求 |
|------|------|
| 浏览器 | Chrome 90+、Firefox 90+、Safari 15+、Edge 90+ |
| 桌面端 | macOS 12+、Windows 10+、Ubuntu 20.04+ |
| 数据库 | PostgreSQL 15+ |

### 4.4 可用性

- UI 语言：中文（简体）
- 无障碍：编辑器支持键盘导航
- 响应式：Web 端支持桌面和平板宽度（≥768px），移动端 MVP 后迭代

## 五、信息架构

### 5.1 页面结构

```
予乐
├── 登录页（/login）
├── 注册页（/register）
├── 主界面（/app）
│   ├── 侧边栏
│   │   ├── 搜索栏（全局搜索）
│   │   ├── 页面树（树形目录）
│   │   │   ├── 根页面列表
│   │   │   └── 子页面（可折叠嵌套）
│   │   ├── 标签筛选区
│   │   └── 新建页面按钮
│   ├── 编辑区
│   │   ├── 面包屑导航（页面层级路径）
│   │   ├── 页面标题（大号可编辑）
│   │   ├── 块编辑器（TipTap）
│   │   │   ├── 块内容区
│   │   │   └── `/` 块类型菜单
│   │   └── 工具栏（块操作、导出）
│   └── 状态栏
│       ├── 保存状态（已保存/保存中.../离线模式）
│       └── 同步状态
└── 404 页面
```

### 5.2 核心交互流程

**页面创建流程：**
```
点击「+ 新建页面」→ 侧边栏出现新页面 → 编辑区显示空白编辑器
→ 输入标题（自动保存）→ 输入内容（自动保存）
```

**块插入流程：**
```
编辑器空行输入「/」→ 弹出块类型选择面板
→ 上下键选择 / 输入过滤 → 回车确认 → 块类型切换
```

**搜索流程：**
```
点击搜索栏 / Ctrl+K → 输入关键词 → 实时显示匹配结果
→ 点击结果 → 编辑区切换到对应页面
```

## 七、接口定义草案

### 7.1 API 基础约定

- **Base URL：** `/api/v1`
- **认证：** `Authorization: Bearer <JWT_TOKEN>`
- **Content-Type：** `application/json`
- **分页：** `?page=1&pageSize=20`
- **错误格式：**
```json
{
  "error": {
    "code": "RESOURCE_NOT_FOUND",
    "message": "页面不存在",
    "details": {}
  }
}
```

### 7.2 接口列表

#### 认证模块

| 方法 | 路径 | 描述 | 认证 |
|------|------|------|------|
| POST | /api/v1/auth/register | 用户注册 | 否 |
| POST | /api/v1/auth/login | 用户登录 | 否 |
| GET | /api/v1/auth/me | 获取当前用户信息 | 是 |

**请求示例 — 注册：**
```json
POST /api/v1/auth/register
{
  "email": "user@example.com",
  "password": "SecurePass123!",
  "name": "用户名"
}

// 成功响应 201
{
  "token": "eyJhbGciOiJIUzI1NiIs...",
  "user": {
    "id": "uuid",
    "email": "user@example.com",
    "name": "用户名",
    "createdAt": "2026-03-20T10:00:00Z"
  }
}
```

#### 页面模块

| 方法 | 路径 | 描述 | 认证 |
|------|------|------|------|
| GET | /api/v1/pages | 获取页面树 | 是 |
| POST | /api/v1/pages | 创建页面 | 是 |
| GET | /api/v1/pages/:id | 获取单个页面 | 是 |
| PUT | /api/v1/pages/:id | 更新页面 | 是 |
| DELETE | /api/v1/pages/:id | 删除页面（软删除） | 是 |
| PUT | /api/v1/pages/:id/move | 移动页面（改父级/排序） | 是 |

**页面数据模型：**
```json
{
  "id": "uuid",
  "title": "页面标题",
  "content": { /* TipTap JSON Document */ },
  "parentId": "uuid | null",
  "order": 0,
  "icon": "📄",
  "tags": ["tag-id-1", "tag-id-2"],
  "createdAt": "2026-03-20T10:00:00Z",
  "updatedAt": "2026-03-20T10:00:00Z",
  "version": 1
}
```

**页面树响应：**
```json
GET /api/v1/pages?tree=true

{
  "pages": [
    {
      "id": "uuid-1",
      "title": "工作笔记",
      "parentId": null,
      "order": 0,
      "children": [
        {
          "id": "uuid-2",
          "title": "项目A记录",

          "order": 0,
          "children": []

      ]

  ]
}
```

#### 搜索模块

**搜索响应：**
```json
{
  "results": [
```

#### 标签模块

| 方法 | 路径 | 描述 | 认证 |
|------|------|------|------|
| GET | /api/v1/tags | 获取所有标签 | 是 |
| POST | /api/v1/tags | 创建标签 | 是 |
| DELETE | /api/v1/tags/:id | 删除标签 | 是 |
| PUT | /api/v1/pages/:id/tags | 更新页面标签 | 是 |
| GET | /api/v1/tags/:id/pages | 获取标签下的页面 | 是 |

#### 文件上传模块

| 方法 | 路径 | 描述 | 认证 |
|------|------|------|------|
| POST | /api/v1/upload/image | 上传图片 | 是 |

**上传请求：** `multipart/form-data`，字段 `file`，支持 jpg/png/gif/webp，最大 5MB。

**上传响应：**
```json
{
  "url": "/uploads/images/uuid-filename.webp",
  "filename": "original.png",
  "size": 102400,
  "mimeType": "image/png"
}
```

#### 导出模块

| 方法 | 路径 | 描述 | 认证 |
|------|------|------|------|
| GET | /api/v1/pages/:id/export?format=markdown | 导出页面 | 是 |
| GET | /api/v1/pages/:id/export?format=pdf | 导出页面 | 是 |

**响应：** 返回文件流，`Content-Disposition: attachment; filename="页面标题.md"`

#### 数据同步模块

| 方法 | 路径 | 描述 | 认证 |
|------|------|------|------|
| POST | /api/v1/sync | 同步本地变更 | 是 |
| GET | /api/v1/sync/changes?since=timestamp | 获取服务端变更 | 是 |

**同步请求：**
```json
POST /api/v1/sync
{
  "changes": [
    {
      "type": "upsert",
      "pageId": "uuid",
      "title": "标题",
      "content": { /* TipTap JSON */ },
      "version": 3,
      "localUpdatedAt": "2026-03-20T10:00:00Z"
    }
  ],
  "lastSyncAt": "2026-03-20T09:00:00Z"
}

// 响应
{
  "synced": ["uuid-1", "uuid-2"],
  "conflicts": [
    {
      "pageId": "uuid-3",
      "localVersion": 3,
      "serverVersion": 4,
      "resolution": "server_wins"
    }
  ],
  "serverChanges": [ /* 服务端自上次同步后的变更 */ ]
}
```

### 7.3 前后端接口约定

- 前端通过 Axios 发起请求，统一拦截器处理 Token 注入和 401 跳转
- 编辑器内容以 TipTap JSON 格式存储，非 HTML 字符串
- 时间统一使用 ISO 8601 格式（UTC）
- ID 使用 UUID v4
- 文件上传使用 FormData

## 九、数据模型概要

### 核心表

```
users
├── id (UUID PK)
├── email (VARCHAR UNIQUE)
├── password_hash (VARCHAR)
├── name (VARCHAR)
├── created_at / updated_at

pages
├── id (UUID PK)
├── user_id (UUID FK → users)
├── title (VARCHAR)
├── content (JSONB)  -- TipTap JSON
├── parent_id (UUID FK → pages, nullable)
├── order (INTEGER)
├── icon (VARCHAR)
├── is_deleted (BOOLEAN, default false)
├── version (INTEGER, default 1)
├── created_at / updated_at

tags
├── id (UUID PK)
├── user_id (UUID FK → users)
├── name (VARCHAR)
├── color (VARCHAR)
├── created_at

page_tags (关联表)
├── page_id (UUID FK → pages)
├── tag_id (UUID FK → tags)

uploaded_files
├── id (UUID PK)
├── user_id (UUID FK → users)
├── filename (VARCHAR)
├── storage_path (VARCHAR)
├── mime_type (VARCHAR)
├── size (INTEGER)
├── created_at
```

### 索引

- `pages(user_id, parent_id)` — 树查询
- `pages(is_deleted)` WHERE `is_deleted = false` — 过滤软删除
- 全文搜索：`pages` 表增加 `search_vector tsvector` 列，GIN 索引
